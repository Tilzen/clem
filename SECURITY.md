# Security Policy

## Reporting Vulnerabilities

We take security seriously and appreciate your help in keeping **clem** secure. If you discover a security vulnerability, please follow these guidelines:

1. **Private disclosure (preferred):** open a [private security advisory](https://github.com/jahwag/clem/security/advisories/new) on the repository. Anything exploitable — a containment bypass, secret exposure, privilege escalation — belongs here, not in a public issue.

2. **Discord:** if you prefer, contact **[@jahwag](https://discord.gg/pR4qeMH4u4)** directly via DM.

3. **Non-sensitive hardening ideas** (defense-in-depth suggestions with no exploitable impact) are welcome as regular [issues](https://github.com/jahwag/clem/issues).

See [docs/threat-model.md](docs/threat-model.md) for what clem does — and deliberately does not — defend against.
